🔄 Strategy 2.1: Smart Room Preservation Flow

1
User Selects Department
User chooses from General Support, Tech Support, or Account Verification
2
Room Status Check
System checks if room exists: 'active', 'left', 'invalid', or none
3
Smart Room Decision
• Active → Direct join
• Left → Request bot re-invitation
• Invalid/None → Create new room
4
Department Switch
Previous room marked as 'left', preserved for future re-entry
5
Room Cleanup
Invalid rooms older than 7 days are automatically cleaned up
Room States Status Tracking
  • ✅ Active: Currently in room
  • 🚪 Left: Room preserved for re-invitation
  • ❌ Invalid: Room no longer exists
  • ➕ None: No room created yet
